## Правила соблюдения

1. **Если опаздываешь — извини в Slack заранее, не молча.**
2. **Если не можешь — назначь делегата** (для Daily / Monday Commit).
3. **Фаундер не лидирует все ритуалы.** Rotation facilitator или PM/CPTO ведёт.
4. **Тайминг жёсткий.** 15 мин = 15 мин. Если не успели — отдельный канал.
5. **Артефакт обязателен.** Без следа в Slack/документе — ритуал умер.

## Адаптация для распределённой команды

Если команда в разных часовых поясах (>3 часов разницы):

| Стандартный ритуал | Адаптация |
|---|---|
| Monday Commit | Async — каждый постит в Slack-thread в свои 10 утра |
| Friday Demo | Loom-видео + Slack-thread для комментариев |
| PSBR | Async через документ + 15-мин синхронный sync |
| Betting Table | 90 мин синхронно в общее окно (выбрать раз в 6 нед) |
| Quarterly Review | Off-site в физической встрече раз в квартал |

---

## Анти-паттерны

❌ Ритуалы добавляются, старые не убираются → команда тонет в meetings.
❌ Daily standup на команде 4 человек → формальность.
❌ Friday Demo превращается в status update без demo → отменить или вернуть demo.
❌ Betting Table занимает 3 часа → жёсткий тайминг, голосование закрытое.
❌ Quarterly Review без off-site (в офисе) → люди не отключаются от текущей работы, плохая ретроспектива.
❌ 6 разных ритуалов в первый месяц внедрения → не выживет.

## Чек-лист: сколько ритуалов реально нужно команде 6-15 чел

**Минимум (обязательно):**
- ✅ Monday Commit (15 мин/нед)
- ✅ Friday Demo (30 мин/нед)
- ✅ PSBR (30 мин/2 нед)
- ✅ Quarterly Review (3 часа/квартал)

**Сильно рекомендуется:**
- ✅ Betting Table (90 мин/6 нед)
- ✅ V2MOM Refresh (1 день/6 мес)

**Опционально (только если Scrum):**
- Sprint Planning + Review + Retro

**Не нужно команде ≤10 чел:**
- Monthly OKR check-in (Friday Demo и Monday Commit покрывают)
- Daily standup (формальность на маленьких командах)

Итого минимум: **2 часа фиксированных встреч в неделю + 3 часа в конце квартала + 1 день в полгода.** Это базовый каркас.
